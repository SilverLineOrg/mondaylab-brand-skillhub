# 信息美学家公众号 HTML 排版

使用这个 Skill，把「信息美学家」专栏文章转换成公众号 HTML 排版稿。

这个 Skill 只负责排版和 HTML 生成，不负责重写选题方向。文章内容和栏目语气使用 `mondaylab-information-aesthetic-column`。

## 输入输出

可接受输入：

- 飞书文档链接、wiki 链接、docx token。
- 本地 Markdown 文件。
- 用户直接粘贴的 Markdown 正文。

默认输出：

- 一份本地 `.html` 文件。
- HTML 适合在浏览器预览，也可以复制正文区域到微信公众号编辑器。
- 如需保留飞书文档里的图片，优先使用飞书 Markdown 导出的图片 URL；如果图片需要重新上传，由用户后续在公众号编辑器里替换。

## 工作流

1. 如果用户给飞书文档链接：
   - 使用 `lark-cli docs +fetch --api-version v2 --doc "<链接>" --doc-format markdown` 获取 Markdown。
   - 保存为临时 `.md` 文件。
2. 如果用户给 Markdown 文件或正文：
   - 直接整理为合法 Markdown。
3. 检查标题层级：
   - 文章标题：`# 标题`
   - 二级视觉标题：`## （1）背景`
   - 三级步骤标题：`### 步骤一：生成「公众号首图」封面`
4. 运行脚本生成 HTML：

```bash
python3 skills/mondaylab-information-aesthetic-wechat-layout/scripts/render_wechat_html.py \
  --input article.md \
  --output article.html
```

5. 交付 HTML 文件路径，并说明可在浏览器打开预览。

## 标题样式

### 二级标题

二级标题对应截图里的大蓝色编号样式。

Markdown 写法：

```markdown
## （1）背景
## （2）了解各平台封面尺寸
## （3）如何用稿定设计做==封面延展==
## （4）为什么==长段落==｜需要==强调重点==
```

生成效果：

- 左侧大号蓝色 `(1)`、`(2)`、`(3)`。
- 右侧黑色大标题。
- 品牌蓝色固定使用 `#0816f1`。
- 二级标题里的 `==文字==` 会变成蓝色圆形重点字样，接近截图里的“封面延展”“长段落”“强调重点”效果。
- 二级标题过长时，可以用 `|` 或 `｜` 手动断成两行；如果标题里有逗号且过长，脚本会去掉逗号并自动换行。
- 二级标题里的重点词尽量控制在 2-5 个字，太长会显得拥挤；重点圆字之间不要使用负间距。
- 二级标题如果分成两行，行高要明显大于默认标题行高，避免蓝色重点字和上一行黑字贴在一起。
- 标题下方自动加一行 `信息美学家Weekly >>>`。

### 三级标题

三级标题对应截图里的次级标题样式。

Markdown 写法：

```markdown
### 步骤一：生成「公众号首图」封面
### 步骤二：生成「飞行社」封面
```

生成效果：

- 居中黑底白字。
- 适合步骤、案例、具体操作说明。

### 四级标题

四级标题保留为正文里的加粗小标题，不做大块视觉样式。

## 图片和截图

Markdown 图片会自动渲染为居中大图：

```markdown
![生成「公众号首图」封面](https://example.com/image.png)
```

如果缺截图，保留占位：

```text
【📷截图：截图说明】
```

脚本会把截图占位渲染成浅灰提示块，方便后续补图。

图片排版规则：

- 图片使用居中 `figure` 容器。
- 图片默认宽度 100%，最大不超过正文宽度。
- 图片使用 4px 圆角。
- 图片阴影使用 mdnice 逻辑：`box-shadow: rgba(170,170,170,0.5) 0px 0px 6px 0px;`，不要使用重卡片投影。
- 图片说明使用 14px 灰色居中小字；没有 alt 时不强行显示说明。

## 表格

Markdown 表格会渲染成适合公众号阅读的浅灰边框表格。

适合放：

- 平台尺寸对比。
- 字段说明。
- 工具清单。
- 步骤对照。

## 排版约束

- 主体宽度按公众号阅读体验控制，默认 `677px`，移动端自动缩放。
- 外层使用 `section#nice`，方便复制到公众号编辑器或 mdnice 类编辑器。
- 字体参考 mdnice 输出，优先使用 `Optima, Microsoft YaHei, PingFangSC-Regular, PingFang SC, serif`。
- 正文使用 16px 字号，1.8em 行高，颜色用 `rgb(74, 74, 74)`。
- 段落使用上 20px、下 8px 的内边距，保持自然的公众号阅读节奏。
- 二级标题前后留大空白，形成截图里的版面呼吸感。
- 三级标题只用于“步骤/案例/操作节点”，不要滥用。
- 列表使用 16px、1.8em 行高，左侧缩进 25px。
- 表格使用浅灰表头、浅灰边框和斑马纹行，方便手机端阅读。
- 文章最后保留固定句：`以上就是本期的全部内容，我们下期见🍻🍻🍻`

## 输出标准

- 生成的 HTML 必须是完整文件，包含 `<!doctype html>`、`<head>`、`<body>`。
- CSS 优先内联在元素上，降低复制到公众号编辑器时丢样式的概率。
- 不要把 HTML 做成复杂网页应用；这是一份公众号排版稿。
- 如果飞书文档里有无法下载或需要用户手动替换的图片，保留原图片链接或截图占位，并在最终说明里提醒。
