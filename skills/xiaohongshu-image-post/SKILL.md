# 小红书图文发布包

使用这个 Skill，把一篇长文或一个选题变成可发布的小红书图文轮播包。

这个 Skill 只负责“小红书图文内容生产”，不负责全平台分发判断。全平台发哪里、发几篇、优先级和 CTA 先由 `content-distribution` 判断；确定要做小红书图文后，再使用本 Skill。

## 内容定位

星期一研究室的小红书图文不做泛流量鸡汤，也不做纯工具截图搬运。

每篇图文都要满足：

1. 有明确读者：知识工作者、内容团队、PM / PMO、AI 工具用户、企业管理者或创作者。
2. 有具体痛点：信息太散、AI 输出无法沉淀、项目没人跟、日报太耗时、知识库不好用等。
3. 有可视化结果：流程图、前后对比、系统截图、字段结构、Demo 页面、工作流地图。
4. 有可保存方法：步骤、字段、结构、Prompt、模板、检查清单。
5. 有轻 CTA：评论关键词、小红书群、私信、留言行业或看主页。

## 默认输出目录

每篇图文独立成一个小包：

```text
xiaohongshu/
  01-result-showcase/
    copy.md
    carousel.md
    cover-prompt.md
    publish-checklist.md
    cover.png
    slide-01.png
    slide-02.png
    slide-03.png
```

如果暂时不生成图片，也要输出：

- `copy.md`
- `carousel.md`
- `cover-prompt.md`
- `publish-checklist.md`

图片文件可以后续补齐。

## 图文类型

优先从以下类型中选择，不要一篇图文塞太多目标。

### 结果展示型

适合展示最终产物。

常见主题：

- 我用 AI 做了一份自动更新的行业报刊
- 一个飞书管理中枢应该长什么样
- AI 输出最后应该沉淀到哪里

结构：

1. 封面：一句强结果。
2. 痛点：原来怎么低效。
3. 最终效果：展示产物。
4. 工作流：展示从输入到输出。
5. 适合谁：列使用场景。
6. CTA：评论关键词或进群看案例。

### 流程拆解型

适合把复杂系统拆清楚。

结构：

1. 封面：一个流程解决一个痛点。
2. 原流程问题。
3. 新流程总览。
4. 人 / AI / Agent / 平台分工。
5. 关键节点。
6. 可复用方法。

### 方法保存型

适合被收藏。

结构：

1. 封面：保存这套结构。
2. 适用场景。
3. 字段 / 模块 / Prompt。
4. 步骤。
5. 检查清单。
6. CTA。

### 使用教程型

适合开源 Skill、Demo 或模板。

结构：

1. 封面：复制这句话即可开始。
2. 安装 / 打开入口。
3. 输入什么。
4. 生成什么。
5. 如何调整到自己的行业。
6. 常见问题。

### 场景迁移型

适合 B 端线索和行业工作流。

结构：

1. 封面：每个行业都可以有自己的系统。
2. 原始行业痛点。
3. 可迁移框架。
4. 行业示例。
5. 你可以怎么开始。
6. 留言行业名称。

## 单篇文件要求

### copy.md

包含：

- 3 个备选标题。
- 正文。
- 话题标签。
- 评论区置顶建议。
- CTA。

小红书正文要轻，不要像公众号长文。

### carousel.md

逐页写清楚：

- 页码。
- 页面标题。
- 页面正文。
- 画面建议。
- 是否需要截图。

默认 6 到 8 页。复杂内容可以到 9 页，但不要超过 10 页。

### cover-prompt.md

包含：

- 封面主标题。
- 副标题。
- 视觉风格。
- 需要出现的元素。
- 不要出现的元素。
- 如果需要生成图片，给出可直接使用的图片生成提示。

封面标题要短，优先让人一眼看懂结果或痛点。

### publish-checklist.md

包含：

- 是否有独立痛点。
- 是否有可视化结果。
- 是否有可收藏方法。
- 是否避免硬导流。
- 是否有评论关键词或群承接方式。
- 是否和同系列其他图文重复。

## 图片生成规则

如果用户要求直接生成图片，优先生成 3:4 或 4:5 竖版图文。

图片风格：

- 清楚、专业、漂亮。
- 偏信息图、流程卡片、截图标注。
- 不做过度网感模板。
- 不使用夸张表情包式设计。
- 不把长段正文塞进图片。

每页图片只承载一个信息点。

## 系列拆解规则

一篇长文可以拆多篇小红书图文，但每篇要有独立角度。

常见系列：

1. 结果展示。
2. 流程拆解。
3. 使用教程。
4. 场景迁移。
5. 成本 / 复盘。
6. 互动征集。

不要把公众号文章按段落机械切成 5 篇。

## 输出要求

当用户给文章或选题时，输出：

1. 推荐拆成几篇小红书图文。
2. 每篇图文的独立主题和目标读者。
3. 每篇图文的目录路径。
4. 每篇的 `copy.md`、`carousel.md`、`cover-prompt.md`、`publish-checklist.md` 内容。
5. 如果要求生成图片，再生成对应 `cover.png` 和 `slide-*.png`。
