# 信息美学家 Weekly：日常提问说明

以后如果你手里有一篇飞书文档，想让我重新生成「信息美学家 Weekly」，可以直接按下面的模板提问。

## 最常用模板

```text
请使用 information-aesthetic-weekly-agent，帮我重新拉取这篇飞书文档，生成信息美学家 Weekly 的 Markdown、栏头图和公众号 HTML，并启动本地预览。

飞书链接：
<这里粘贴飞书 wiki/docx 链接>
```

这个模板会默认完成：

- 用飞书 CLI 重新拉取文档正文、图片和图片描述。
- 生成本期 Markdown。
- 调用栏头图 skill 生成顶部杂志栏头图。
- 把二级标题生成 `1080×500` 的图片标题。
- 把栏头图和二级标题图上传到仓库里的 `assets/information-aesthetic-weekly/`。
- 在 Markdown 和 HTML 里使用 GitHub raw 图片地址，方便复制到公众号。
- 生成公众号 HTML。
- 启动或复用本地预览服务。

## 带期数和文件名的模板

如果你已经知道期数和文章 slug，可以这样写：

```text
请使用 information-aesthetic-weekly-agent，帮我重新拉取这篇飞书文档，生成信息美学家 Weekly 的 Markdown、栏头图和公众号 HTML，并启动本地预览。

飞书链接：
<这里粘贴飞书 wiki/docx 链接>

期数：013
slug：article-013-your-topic
```

如果不写期数和 slug，我会根据文档标题和已有文件尽量推断，但正式发布前建议手动给一下。

## 只重新渲染 HTML

如果飞书内容没有变，只是想调整公众号样式，可以这样写：

```text
请使用 information-aesthetic-weekly-agent，基于当前 Markdown 重新生成公众号 HTML，并启动本地预览。
```

## 重新拉飞书，但不重做栏头图

如果只是正文有小改动，栏头图不想变：

```text
请使用 information-aesthetic-weekly-agent，重新拉取这篇飞书文档并生成 Markdown 和公众号 HTML，栏头图先不要重做。

飞书链接：
<这里粘贴飞书 wiki/docx 链接>
```

## 飞书文档里的标注方式

- `|` 或 `｜`：手动指定二级标题视觉折行。
- `==重点==`：手动指定蓝色强调词。
- `[结语：]`：只给排版系统看的删除标记，不进入二级标题图。
- 飞书下划线：会尽量保留到公众号 HTML。
- 图片下方的手写说明：会作为图片说明保留。

如果没有写 `==重点==`，我会自动从标题里找一个短重点词。

## 发布前常用检查

生成后，你可以继续这样提：

```text
帮我重新拉取飞书文档，重新渲染一遍效果。
```

```text
帮我把当前效果提交并推送到仓库。
```

```text
帮我检查一下复制到公众号后，标题、图片、callout 和正文间距有没有容易丢样式的地方。
```

## 注意

栏头图和公众号正文是两条独立支线。栏头图只根据飞书文档主题提炼文案，不混用公众号二级标题的折行、蓝色强调、删除问号等规则。
